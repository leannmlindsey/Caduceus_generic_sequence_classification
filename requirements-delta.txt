#   pip install --no-cache-dir --no-build-isolation causal-conv1d>=1.4.0

# THREE RISK POINTS that will likely need iteration (validate each with the
# test script + a 1-genome inference, and for fine-tuning a 1-seed dry run):
#
#  1. mamba-ssm 1.x -> 2.x API. The repo was written against mamba-ssm 1.2.0.
#     2.x is required for sm_90 but moved/renamed some internals. If `import
#     caduceus` (the model code) fails after the bump, the fix is small import
#     shims in the caduceus modeling files, NOT downgrading (1.2.0 won't run on
#     GH200). causal-conv1d 1.4+ pairs with mamba-ssm 2.2+.
#
#  2. flash-attn is REQUIRED (corrected). Although Caduceus-PS itself is pure
#     Mamba, the repo's src/models/sequence/long_conv_lm.py imports
#     flash_attn.modules.{block,mha,mlp,embedding} + utils.generation.GenerationMixin
#     UNCONDITIONALLY, and dna_embedding.py imports long_conv_lm, so the model will
#     not import without flash-attn. It must be built from source for sm_90 (the
#     Biowulf 2.5.6 pin won't build on torch 2.12 — use current). See
#     build_flashattn_delta.sbatch / step 7b in build_caduceus_env.sh.
#     Do NOT uninstall triton: mamba-ssm 2.x uses torch's bundled Triton 3.x.
#
#  3. pytorch-lightning for FINE-TUNING only. Inference/embedding load the ckpt
#     with torch.load (src/inference.py), so they do NOT need Lightning. But
#     `python -m train` (fine-tuning) is built on the Caduceus Hydra+Lightning
#     framework pinned to pl 1.8.6. Try pl==1.8.6 on py3.11 first; if its old
#     API breaks under modern torch, bump to a 2.x and expect a few train.py /
#     Trainer call-site fixes. Inference can proceed even while this is sorted.
# ----------------------------------------------------------------------------
#
# torch is installed SEPARATELY (above), not pinned here — the correct wheel is

# on py3.11). mamba-ssm + causal-conv1d are also installed separately (source
# build, --no-build-isolation so they see the already-installed torch).
